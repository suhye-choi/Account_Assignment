### MainQuest 01

- 주제 : 은행계좌 만들기
- 🧩 파이썬 은행 시스템 기획서 (최신 코드 기반 업데이트 버전)
    
    **[로그인 편]**
    
    - SC 은행 어플에 들어감(최초화면)
    - 개인인증을 한 뒤 로그인 완료
    
    ---
    
    **[계좌 개설편]**
    
    - input : 통장별명, 비밀번호 입력
    - input : 비밀번호 재입력
    - if : 초기 비밀번호와 재입력 비밀번호 일치 시
        - 다음 플로우로 이동
    - else : 미일치 시
        - 비밀번호 재입력으로 이동
    - print() : 계좌 개설 후 해당 정보를 방문객이 볼 수 있게 제출(영수증 같이)
        - 은행이름 : SC 은행
        - 통장별명 : input 값 그대로
        - 계좌번호 3-2-6 자리 랜덤 발생
    
    **처리 결과**
    
    - 신규 계좌(`Account` 객체) 생성
    - 계좌번호 : 3-2-6 자리 랜덤 생성
    - 초기 잔고 : 0원
    
    **[메인화면]**
    
    조건
    
    - 최초 실행 시 계좌가 없으면 무조건 계좌 개설 진행
    - 이후에는 계좌 목록이 화면에 표시됨
    - input: 계좌 선택(**kwargs)_enurmerate(””, start=1) 로 하고 계좌가 늘어날 수록 늘어남, 그리고 계좌 선택 수는 넘버링에 따라 달라질 수 있음
        - 여기에서는 기존 계좌와 신규 계좌를 보여줘야 함(메뉴판 참고)
            - 계좌별명 : 123-45-678900/ 잔고: 10000원
            - 계좌별명 : 위에서 랜덤으로 발생한 숫자/ 잔고:0원
    - input : 계좌 내역 확인
        - 선택지 : 1. 계좌 내역보기, 2.입금, 3.출금, 4.신규 계좌 개설 5. 종료
    
    **[입금 시]**
    
    - 메인화면에서 2. 입금 선택 시 해당 함수 활성화
    - input : 어떤 통장에서 돈을 옮길까요?
        - list 호출, 해당 리스트는 계좌 개설에 만들어진 계좌의 리스트+ 인덱싱을 통해 사용자가 선택할 수 있도록 함.
        - 예시 : 1.[오늘도부자통장]+계좌번호
        - 계좌 선택 후 다음 프로세스
    - input : [통장별명]으로 얼마나 돈을 옮길까요?
        - if 문
            - 숫자로만 받을 수 있음+원
            - 잔액은 default 10만원으로 보여주기→ 만약 신규 계좌로 이동 시 해당 잔액은 차감됨
            - 다음 선택
        - else 문
            - 잔액 이상 기재, 확인 시 아래 문구 출력
            - “잔액이 부족해요, 다시 확인해주세요.” return = input 으로
        
    - ”내 입출금 통장으로 “~원”을 옮길까요?” y/n
        - y 선택 시
            - 비밀번호 입력 요구
            - 옮겼을 때 pause 1초 후 안내 문구 “”내 입출금 통장으로 “~원”을 옮겼어요”
            - input : 메인 화면으로 돌아가기 , 혹은 추가 입금하기 중 선택
        - n 선택 시
            - 메인화면으로 돌아갑니다.
            - 1 pause 후 return = 메인화면
    
    **[이자 지급]**
    
    - 조건 : [입금] 이 5회, 10회, 15회 등 5배수로 발생 시점에서 이자 1% 지급
    - 이자 지급 시점 : 5회의 [입금]이 완료된 상태에서 자동 출력
    - 이자 지급 멘트
        - 축하합니다. [고객명]님의 소중한 계좌에 이자를 쏙 넣어드렸어요.
        - 이자= 총금액 = 잔고 x  0.01% / 총 잔고
        - input : 확인 선택 시 메인으로 이동
    
    **[출금 시]**
    
    - 메인화면에서 3.출금 선택 시  해당 함수 활성화
    - input : 얼마나 옮길 지 기입
        - 잔고 이상으로는 출금할 수 없음
        - 입력 된 숫자+원
        - input : 확인 / 다시 입력하기
        - 입력 후 “[계좌명]에 [~원]을 옮길까요?” 출력, 확인/ 안 옮김 선택
        - input : 비밀번호 입력
            - if : 비밀번호가 일치하면→ 확인으로
            - else : 비밀번호 미일치시 → 재입력 요청
        - 확인 : 안전하게 옮겼어요.
        - input : 메인 화면으로, or 추가 출금
    
     **[내역확인]**
    
    - input : 계좌 선택
    - 입금과 출력, 이자지급 히스토리 리스트업
    - 하나의 리스트에는 입금자: 최수혜/ 이자:SC 은행, 발생 금액, 최종 잔액, 날짜 가 한세트임
